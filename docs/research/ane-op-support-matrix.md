# ANE Compiler Op Support Matrix

Probed against Apple's private `_ANEInMemoryModel` compiler on macOS (Apple Silicon).
Generated by `cargo run -p ironmill-ane --example ane_op_probe`.

> **Note:** Results may vary by macOS version and chip generation. These results were
> obtained using the `program(1.3)` / `func main<ios18>` MIL text format with
> `[1,C,1,S]` tensor layouts (ANE-native NCHW). Some ops that fail standalone may
> work in specific contexts (e.g., as part of a fused pattern).

## Unsupported Ops (27 confirmed failures)

### Elementwise Math (rejected)
| Op | Status | Workaround |
|---|---|---|
| `log` | ❌ | Decompose: `log(x) = log2(x) / log2(e)` via exp2 inverse, or CPU fallback |
| `rsqrt` | ❌ | Decompose: `pow(x, -0.5)` or `real_div(1, sqrt(x))` |
| `inverse` | ❌ | Decompose: `real_div(1, x)` |
| `neg` | ❌ | Decompose: `mul(x, -1)` or `sub(0, x)` |
| `mod` | ❌ | Decompose: `sub(x, mul(floor_div(x, y), y))` |
| `sin` | ❌ | CPU fallback or polynomial approximation |
| `cos` | ❌ | CPU fallback or polynomial approximation |

| `asin` | ❌ | CPU fallback |

## TurboQuant / PolarQuant Feasibility on ANE

### QJL Sign Extraction — ✅ FULLY FEASIBLE
All required ops are supported:
- `sign` — direct sign extraction
- `greater` / `less` — comparison to zero
- `cast fp16↔bool` — type conversion
- `select` — conditional selection
- `reduce_sum` — dot product computation

### Hadamard Rotation — ✅ FEASIBLE
- `matmul` — matrix multiplication for rotation

### PolarQuant Normalization — ✅ FEASIBLE
- `reduce_sum` — sum of squares
- `pow` — inverse square root (`pow(x, -0.5)`)
- `mul` — scaling
- `layer_norm` — direct normalization

### Scalar Quantization/Dequantization — ✅ FEASIBLE
- `mul`, `add`, `clip`, `round` — affine quantization
- `cast fp16→int8` — type conversion
- `dequantize` — INT8 dequantization

### LUT Dequantization — ⚠️ NEEDS INVESTIGATION
- `gather` failed standalone probe but may work with specific tensor shapes
  or as part of `constexpr_lut_to_dense` patterns

### Cache Mutation — ❌ REQUIRES CPU FALLBACK
- All scatter variants rejected
- Must use CPU interception at ANE sub-program boundaries
- Pattern: ANE computes quantized K/V → CPU writes to cache → ANE reads back

## Notes

### Ops confirmed by maderix/ANE project
The [maderix/ANE](https://github.com/maderix/ANE) project independently confirms
these ops work in production ANE training kernels:
- `conv`, `matmul`, `add`, `mul`, `sub`, `sigmoid`, `softmax`, `reshape`,
  `transpose`, `reduce_sum`, `pow`, `concat`, `slice_by_size`
- `constexpr_affine_dequantize` for INT8 weights
- `quantize`/`dequantize` for INT8 activation caching (1.88× throughput)

### Common decomposition patterns
Many "unsupported" ops have simple decompositions into supported ops:
- `neg(x)` → `sub(0, x)` or `mul(x, -1)`
- `rsqrt(x)` → `pow(x, -0.5)`
- `inverse(x)` → `real_div(1, x)`
- `logical_and(a, b)` → `cast(mul(cast(a, fp16), cast(b, fp16)), bool)`

### Shape constraints
Some ops may fail due to shape constraints rather than being fundamentally
unsupported. The ANE has specific alignment requirements:
- Input channels should be aligned to 32
- Conv kernels ≤ 16×16
- Tensor layout must be `[1, C, H, W]` (NCHW)
